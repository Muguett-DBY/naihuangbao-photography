# 完整方案：修复页面切换 + 填充真实内容

## 问题诊断

### 页面切换空白的根本原因（已部分修复，需彻底排查）

已修复的：
- `.section-shell` 默认 `opacity: 0` → 已改为 `opacity: 1`
- `.why-card` / `.package-card` / `.process-step` 默认隐藏 → 已改为可见
- GSAP `fromTo` 隐藏状态 → 已移除

仍需排查的：
1. `AnimatePresence mode="wait"` + `Suspense` 组合可能导致新页面未渲染就卸载
2. `useGsapPageEffects` 的 `guardRef` 在 StrictMode 下可能阻止动画执行
3. 某些组件（Gallery、WhyChooseUs）使用 `Section` 组件，其 `useInView` 可能不触发
4. CSS 中可能还有其他 `opacity: 0` 规则

### 修复方案

**核心原则：所有内容默认可见，动画只做增强。**

1. 简化 `AnimatePresence` → 移除 `mode="wait"`，改为同时渲染
2. `Section` 组件默认添加 `is-visible`
3. 排查所有 CSS 中的隐藏规则
4. 移除所有 GSAP/JS 依赖的内容显示逻辑

## Phase 1: 彻底修复页面切换空白

### 1.1 RootLayout 简化

**文件**: `src/layouts/RootLayout.tsx`

移除 `AnimatePresence mode="wait"`，改为简单的路由切换：

```tsx
// 移除 AnimatePresence 包装，直接渲染 Outlet
<motion.main
  key={location.pathname}
  id="main-content"
  initial={{ opacity: 0 }}
  animate={{ opacity: 1 }}
  transition={{ duration: 0.3 }}
>
  <Outlet />
</motion.main>
```

### 1.2 Section 组件默认可见

**文件**: `src/components/Section.tsx`

移除 `useInView` 依赖，直接添加 `is-visible`：

```tsx
<section id={id} className="section-shell is-visible">
```

### 1.3 CSS 清理

**文件**: `src/styles/base.css`

确认所有内容相关 CSS 规则默认 `opacity: 1`。

### 1.4 useGsapPageEffects 简化

**文件**: `src/hooks/useGsapPageEffects.ts`

移除所有可能隐藏内容的逻辑，只保留纯装饰性动画。
