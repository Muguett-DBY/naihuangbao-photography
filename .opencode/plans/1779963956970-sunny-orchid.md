# 超级大升级计划

## 总览

将网站从 8.5/10 提升到 9.5/10，分 5 个独立可部署阶段。

---

## Phase 1: UI 视觉升级（卡片重设计）| 复杂度 M

**目标**：二级页面卡片对齐首页设计语言，消除视觉割裂感

### 修改文件

**`src/styles/pages.css`** — 重设计 4 种卡片：
- 替换扁平 `background: var(--card-bg)` → 多层渐变（同 `.package-card`）
- 替换 `box-shadow: none` → `var(--paper-inset), var(--warm-shadow)`
- 添加 `::before` 装饰性径向渐变圆
- `border-radius` 16px → 22px
- Hover: `translate3d(0, -4px, 0)` + `var(--warm-shadow-strong)` + `cubic-bezier(0.22, 1, 0.36, 1)`
- 添加 `backface-visibility: hidden` GPU 合成
- `.workshop-cover` / `.course-cover` 添加 `::after` 暗角遮罩

**`src/styles/sections.css`** — 添加 cute 主题覆盖：
- `:root[data-mood="cute"] .course-card` 等，`border-radius: 28px`

**`src/pages/CoursesPage.tsx`** / **`PresetsPage.tsx`** / **`WorkshopsPage.tsx`** / **`ShopPage.tsx`**：
- 卡片 Link 外层包裹 `position: relative` div 支持 `::before`

---

## Phase 2: 内容补全 + 数据修复 | 复杂度 S

**目标**：修复数据不一致和 i18n 缺口

### 修改文件

**`src/data/packages.ts`** — 统一设备信息：
- `serviceAddOns.equipment` 列表与 i18n locale 文件对齐

**`src/i18n/locales/*.json`**（4 个文件）— 添加缺失翻译：
- `serviceDetails.equipment.list`（设备列表）
- `chat.*` 系列（聊天组件完整翻译）

**`src/components/PublicChatWidget.tsx`** — 中文硬编码 → `t()`：
- `starterPrompts` → `t("chat.prompts")`
- `welcomeMessage` → `t("chat.welcome")`
- `aria-label` → `t("chat.ariaLabel")`
- `placeholder` → `t("chat.placeholder")`

## Phase 3: i18n 完善（Admin 多语言）| 复杂度 M

**目标**：Admin 后台支持 4 语言

### 修改文件

**`src/components/admin/AdminShell.tsx`**：
- Tab 标签、登录消息、标题、登出按钮全部 `t()` 化
- `adminTabItems` 改为函数或 `useMemo`

**`src/components/admin/*.tsx`**（所有 Tab 组件）：
- 审计并替换所有硬编码中文

**`src/i18n/locales/*.json`** — 添加 `admin.*` 完整翻译段

## Phase 4: 性能优化 | 复杂度 L

### 4A: Three.js 代码分割
- **`vite.config.ts`** — 添加 `three-vendor` manual chunk
- **`src/components/Gallery.tsx`** — PhotoWall3D 改为 `React.lazy()` 懒加载
- **`src/pages/GalleryPage.tsx`** — 包裹 `<Suspense>`

### 4B: Canvas 动画节流
- **`src/components/CanvasParticles.tsx`** — 30fps 移动端限制 + `document.hidden` 暂停
- **`src/components/PhotoWall3D.tsx`** — `frameloop="demand"` + IntersectionObserver

### 4C: CSS 精简
- **`src/styles/pages.css`** — 提取 `.card-base` 公共样式减少重复
- 合并分散的 grid 定义

## Phase 5: 新功能（支付/通知/用户/视频）| 复杂度 XL

分 4 个子阶段，每个独立部署：

### 5A: 支付集成（Cloudflare Payments 或 Stripe）
- 新建 `src/components/PaymentForm.tsx`
- 新建 `functions/api/payment/create-intent.ts`
- 新建 `functions/api/payment/webhook.ts`
- 修改 `BookingPage.tsx` / `CourseDetailPage.tsx` / `WorkshopDetailPage.tsx`

### 5B: 邮件/通知系统
- 新建 `functions/api/notifications/send.ts`
- 新建 `src/hooks/useNotification.ts`
- 修改 `BookingModal.tsx` / `WorkshopsPage.tsx`

### 5C: 轻量用户系统
- 新建 `functions/api/auth/*`（D1 魔法链接或密码）
- 新建 `src/hooks/useAuth.ts` + `src/components/AuthProvider.tsx`
- 修改 `Header.tsx`（用户菜单）/ `CourseDetailPage.tsx`（内容门控）

### 5D: 视频课程支持
- 新建 `src/components/VideoPlayer.tsx`（原生 `<video>` 或 Plyr）
- 修改 `CourseDetailPage.tsx`（视频模块渲染）

## 不做的事

| 避免 | 原因 |
|------|------|
| CSS → Tailwind | 重写所有组件，风险巨大 |
| 替换 Three.js | 核心差异化功能 |
| 添加 Redux/Zustand | 状态简单，过度工程 |
| 自定义 CMS | Admin 已够用 |
| SSR/Next.js 迁移 | SPA + Cloudflare 工作良好 |
| 原生移动 App | PWA 已安装 |
| 自建分析系统 | GA/Plausible 足够 |

## 依赖关系

```
Phase 1 (UI 升级)  ─────────────────┐
Phase 2 (内容修复)  ──┐              │
                      ├──── Phase 5 (新功能)
Phase 3 (i18n 完善) ─┘              │
Phase 4 (性能优化)  ────────────────┘
```

- Phase 1/2/3/4 **互相独立**，可单独部署
- Phase 3 依赖 Phase 2（聊天 i18n key 需先存在）
- Phase 5 依赖 Phase 2+，但可在 Phase 2 后开始
