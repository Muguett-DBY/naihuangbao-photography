# 修复方案：移动端导航菜单被遮挡/不显示

## 问题根因

虽然桌面浏览器模拟移动端viewport时导航菜单正常，但在真实iOS Safari上有4个关键问题导致菜单不可见或不可用：

### Critical Issue #1: `body.nav-lock` 缺少 `position: fixed`
**文件**: `src/styles/base.css:409-412`
```css
body.nav-lock {
  overflow: hidden;       /* iOS Safari: 使 body 成为滚动容器 */
  touch-action: none;     /* 禁止所有触摸交互 */
}
```
在iOS Safari上，`overflow: hidden` 会让 `position: fixed` 子元素相对于 `body` 定位而非视口。由于 `body` 本身没有固定尺寸，`.nav-menu`（`position: fixed`）被定位到 body 的实际滚动区域之外 → **完全不可见**。

**修复**: 添加 `position: fixed; width: 100%; height: 100%`。

### Critical Issue #2: `backdrop-filter` + `position: fixed` 在 iOS Safari 上渲染失败
**文件**: `src/styles/sections.css:1279-1280`
```css
.nav-menu {
  backdrop-filter: blur(16px) saturate(1.1);  /* iOS Safari 渲染bug */
}
```
iOS Safari 有已知bug：`backdrop-filter` + `position: fixed` 导致元素合成失败，表现为元素透明/不可见。同时父级 `.site-nav` 也有 `backdrop-filter`，复合使问题加剧。

**修复**: 移动端移除 `.nav-menu` 的 `backdrop-filter`，使用纯色背景替代。

### Critical Issue #3: `touch-action: none` 阻止导航菜单滚动
**文件**: `src/styles/base.css:411`
`touch-action: none` 传播到所有子元素，导致 `overflow-y: auto` 的 `.nav-menu` 无法通过触摸滚动。

**修复**: 在 `.nav-menu.is-open` 上添加 `touch-action: auto` 覆盖。

### Issue #4: `.nav-menu a` 缺少 `position: relative`
flex 子元素的 `z-index` 需要 `position: relative` 才能正确生效，否则链接可能渲染在 `::before` 卡片后面。

**修复**: 添加 `position: relative` 到 `.nav-menu a`。

### Issue #5: CSS 重复定义
移动端 `@media (max-width: 760px)` 内有重复的 `.hamburger`、`.brand-mark`、`.lang-toggle` 等规则，后者覆盖前者。

**修复**: 清理重复的CSS规则。

## 要修改的文件

### 1. `src/styles/base.css` — 修复 body.nav-lock
- `body.nav-lock` 添加 `position: fixed; width: 100%; height: 100%;`

### 2. `src/styles/sections.css` — 多个修复
- `.nav-menu` 移动端移除 `backdrop-filter`/`-webkit-backdrop-filter`
- `.nav-menu a` 添加 `position: relative`
- `.nav-menu.is-open` 添加 `touch-action: auto`
- 清理重复的CSS规则（`.hamburger`、`.brand-mark` 等）

## 验证方案
- 启动 `npx vite`，用 Playwright 设置 375x812 viewport
- 点击 hamburger 按钮
- 截图 + 获取 computed styles 确认 `.nav-menu` 可见
- 确认 `body.nav-lock` 有 `position: fixed`
- 确认 `.nav-menu` 无 `backdrop-filter`
- 运行 `npm run lint` + `npm test` + `npx vite build`
